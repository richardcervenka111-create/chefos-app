Najdôležitejší záver expert review

v1 zoznam bol o produkte. Chýbali v ňom tri kategórie, ktoré potápajú startupy častejšie než chýbajúce funkcie: (1) právne základy — IP voči zamestnávateľovi, s ktorým máš zároveň spor, je najvážnejšie jediné riziko v celom projekte; (2) inžinierska disciplína — žiadny Git, žiadny staging, SQL rovno na produkciu, jeden súbor bez zálohy; (3) validácia peniazmi — trial bez vopred definovaného úspechu a bez otázky „zaplatil by si?" je len demo pre kamarátov.

Poradie na najbližších 14 dní

1. Zálohy DB + test obnovy → 2. IP kontrola novej pracovnej zmluvy → 3. Resend doména → 4. Staging projekt → 5. Deploy claude-proxy + spustiť zvyšné SQL 46–54 → 6. Regresný checklist naživo na reálnom zariadení → 7. Kritériá úspechu trialu → potom trial. (Git, API kľúč na server, realtime, analytika, error tracking, invite expirácia a audit log sú už hotové. Aj kritický RLS recursion bug z db/53 — nájdený a opravený v ten istý deň, db/55 už spustené a naživo overené.)
